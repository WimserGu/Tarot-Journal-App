## 0. 本轮范围说明

### 0.1 与现有信息架构的差异

本轮需求引入了三个需要后续同步到信息架构的变化：

| 最新需求         | 现有信息架构                     | 本线框图采用的处理                                                     |
| ---------------- | -------------------------------- | ---------------------------------------------------------------------- |
| 单独的“首页”     | 议题列表即默认主页面，不单设首页 | 增加轻量首页，用于草稿、最近问题和快速继续；完整管理仍在议题列表       |
| 允许输入临时问题 | 每条记录必须归属于固定问题       | 临时问题仍必须归属于长期议题，但不进入固定问题统计；之后可转为固定问题 |
| 支持暂存草稿     | 不承诺关闭 App 后恢复未保存草稿  | 增加可恢复草稿；草稿不进入时间线和统计                                 |

因此，本线框图暂定底部导航为：

```text
首页 | 议题 | ＋记录 | 回顾
```

“设置”继续通过右上角账户入口进入。是否保留独立首页，应在线框图测试后决定；如果首页与议题列表被用户认为重复，应合并回三入口结构。

### 0.2 临时问题的产品边界

- 临时问题必须选择一个长期议题，避免形成完全孤立的单次占卜。
- 临时问题只保存本次问题文本，不出现在固定问题列表。
- 临时问题记录出现在议题近期记录和回顾页中。
- 临时问题不进入“同一固定问题”的时间线和数据分析。
- 用户可从记录详情将临时问题保存为固定问题；转换后，该记录可以成为新固定问题的第一条历史记录。

### 0.3 低保真符号

```text
[按钮]       可点击命令
<选择器>     点击后进入选择或展开选项
[ 输入框 ]   文本输入
(● / ○)      单选状态
[x / ]       复选状态
≡            拖动排序手柄，同时提供上移 / 下移操作
⋮            更多操作菜单
```

### 10.1 正式保存规则

正式保存至少需要：

- 一个长期议题。
- 一个固定问题，或非空的临时问题文本。
- 日期和时间。
- 至少一个完整牌条目。
- 每个牌条目都有牌名和正逆位；位置名称可以为空。

个人解释与后续反馈可以稍后补充，以保留快速录入能力。

### 10.2 草稿规则

- 草稿允许字段不完整，也允许牌条目尚未选择牌名。
- 草稿不出现在时间线、数据分析或正式记录数量中。
- 草稿显示在首页，带所属议题或“未选择议题”的状态。
- 再次打开草稿时恢复牌顺序和所有已输入字段。
- 离开有修改但尚未暂存的页面时，提供“暂存并退出”“继续编辑”“放弃修改”。

### 10.3 排序与问题切换规则

- `≡` 支持拖动排序，同时提供“上移 / 下移”以保证可访问性。
- 排序后位置名称与牌一起移动，避免牌和位置错位。
- 切换长期议题会清空当前固定问题选择；如果已经输入牌面，需要先确认。
- 从固定问题切换为临时问题时，不删除已录入牌，但说明该记录将不进入固定问题统计。
- 编辑已保存记录并移除已有牌时，最终保存前需要二次确认。

### 18.1 新用户建立长期结构

```text
→ WF-10 选择一张牌

→ WF-11 记录详情
```

### 18.3 临时问题记录

```text
底部“＋记录”
→ WF-09 选择长期议题

→ 添加牌并保存
→ WF-11 显示“临时”标识
→ 可选择“保存为固定问题”
→ WF-08 完成模板设置
→ WF-12 成为该问题的第一条时间线记录
```

### 18.4 草稿恢复

```text
WF-09 输入部分内容
→ “暂存草稿”
→ WF-03 首页草稿区
→ “继续”
→ WF-09 恢复全部字段和牌顺序

```

### 18.5 同一问题复盘

```text
WF-06 固定问题列表
→ WF-12 时间线
→ WF-11 查看某次完整记录
→ 返回 WF-12
→ WF-13 数据分析
→ 用样本量、重复牌、花色和正逆位核对长期变化
```

## 19. 低保真验收清单

- 欢迎页能让首次用户理解“议题 → 固定问题 → 记录”，而不是期待单次 AI 解牌。
- 首页和议题列表职责不同：前者用于快速继续，后者用于完整管理。
- 新增记录始终优先要求选择长期议题。
- 用户可以选择固定问题，也可以输入隶属于议题的临时问题。
- 临时问题、草稿和正式固定问题记录在界面上有明确区别。
- 记录可添加任意数量牌，每张牌都有独立的牌名、正逆位和可选位置。
- 牌条目支持拖动和上移 / 下移，排序时牌与位置一起移动。
- 草稿允许不完整，但不会污染正式时间线和统计。
- 正式保存失败时不清空用户输入。
- 时间线与数据分析始终显示所属议题和固定问题。
- 数据分析明确显示样本量与日期范围，不使用预测性语言。
- 删除正式记录、移除已保存牌和放弃未保存修改受到保护。

## 20. 建议优先测试的问题

1. 用户是否理解首页与议题列表的区别；如果不能，应将两页合并。
2. 临时问题是否帮助快速记录，还是削弱了固定问题的核心价值。
3. 一张牌路径能否在不遗漏正逆位和归属问题的前提下，于一分钟内完成。
4. 多牌排序时，用户是否能始终确认牌、位置和顺序的对应关系。
5. 用户是否能区分草稿、正式记录和尚未补充反馈的记录。
